05 — Interaction & layout specs Documentation frame

Component & layout rules

Aligns localOCR UI with setcalculators.com patterns so the product feels like a calm, trustworthy browser tool — not a generic AI dashboard.

Inspiration mapping

Dual radial page wash · pill nav · surface cards · indigo primary · cyan accent · max-width: 1100px content rhythm · privacy copy near actions.

Workspace grid

200 | 1fr | 340 desktop columns. Page rail left, canvas center, results right. Status bar sticks to canvas bottom with cancel always visible.

Confidence UI

Mono scores; green ≥0.9, amber 0.75–0.9, muted below. Hovering a result line highlights the matching bbox (cyan).

Buttons

Primary = solid indigo pill + soft indigo shadow. Secondary = white surface + 1.5px slate border. Never all-caps.

Privacy affordances

Persistent “On-device” pill with green pulse dot on every chrome level. Export screen repeats “Nothing left this device”. Legal / info links live in the site footer (Privacy, Terms of use, About us) — not the top bar.

Top bar vs footer

Landing top bar: On-device · WebGPU ready · How it works only. No History chrome. Footer: Privacy · Terms of use · About us · How it works.

Empty → active

Landing is a single focused drop card (like setcalculators’ calculator card). Workspace only appears after a file is accepted — no empty multi-pane clutter.
